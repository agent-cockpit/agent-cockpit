## Critical Gaps (Blockers)

### Gap 1: ApprovalInbox is a UI stub — APPR-01, APPR-02, APPR-03, APPR-04

`packages/ui/src/components/panels/ApprovalInbox.tsx` is a 7-line placeholder stub rendering "Pending approvals will appear here in Phase 3." The daemon correctly registers, classifies, and persists approvals. `approval_request` events reach the Zustand store and increment `pendingApprovals` counters on `SessionCard` and `AgentHoverCard`. But:

- The inbox panel never renders approvals from the store
- `useSessionEvents.ts` module-level `ws` variable is not exported — no `ws.send()` callable from UI
- Users cannot see, inspect, approve, deny, or always-allow any pending approval

**Fix:** Implement ApprovalInbox with store subscription + expose sendMessage from useSessionEvents.

### Gap 2: Notifications not wired to UI — NOTIF-01, NOTIF-02

`notificationHelpers.ts` (with `buildNotificationPayload` and `shouldNotifyOS`) lives in `packages/daemon/src/notifications/` — the wrong package for browser import. The `useSessionEvents.ts` `onmessage` handler only calls `applyEvent(event)` with no notification logic. No in-app toasts, no `new Notification()` calls anywhere in the UI.

**Fix:** Move or duplicate notification logic to packages/shared or packages/ui; call from onmessage handler on approval_request and session_end.

### Gap 3: SearchBar not mounted — HIST-01

`packages/ui/src/components/search/SearchBar.tsx` exists with correct `GET /api/search?q=` fetch and is unit-tested in isolation. It is imported nowhere except its own test file. `HistoryPage.tsx` renders session list + ComparePanel but no SearchBar. The `/api/search` daemon endpoint is correctly implemented with FTS5.

**Fix:** Import and render SearchBar in HistoryPage.tsx above the session list.

## Integration Issues (Non-blocking)

### Issue 1: LaunchSessionModal relative URL — SESS-02

`LaunchSessionModal.tsx:29` uses `fetch('/api/sessions', ...)`. All other UI fetches use `http://localhost:3001/...`. No `server.proxy` in `vite.config.ts`. Request will 404 in development. Quick fix: change to absolute URL matching other fetches.

### Issue 2: Missing project/recency filter in live session list — SESS-04

`SessionFilters.tsx` has provider, status, and workspace search. No project-name discrete select or recency (e.g. "last hour / today / this week") filter. These exist only in `HistoryPage` for historical sessions. Requirement says "filter the session list by project, provider, status, and recency."
