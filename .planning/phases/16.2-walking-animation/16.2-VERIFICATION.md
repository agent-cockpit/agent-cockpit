### Notable Deviation: WALK_FRAME_COUNT = 8

Plan 01 specified `WALK_FRAME_COUNT = 4`, but the implementation exports `WALK_FRAME_COUNT = 8`.

The player sprite sheet (`astronaut-sheet.png`) has 8 columns per row (consistent with `STATE_ROW_OFFSET` values of 0, 8, 16, 24 — indicating 8 directions × 8 frames per animation state). Using `4` would have played only half the available walk frames. The implementor correctly adjusted to match the actual sprite sheet.

The test "col formula wraps from frame 3 back to frame 0" still passes but now tests `3 % 8 = 3`, not a true wrap-at-4 assertion. This is mathematically correct but the test name is now misleading. This is a warning-level concern only — the walk animation goal is met.

### OfficePage Player Row-Switching

OfficePage L115 adds extra logic not in Plan 02:
```
const stateOffset = gameState.player.animTime > 0 ? STATE_ROW_OFFSET.blocked : STATE_ROW_OFFSET.idle
```

This switches the player sprite between the idle row-group (row offset 0) and the blocked row-group (row offset 8) based on whether the player is moving. This was not specified in Plan 02. Whether this maps to the correct "walking" vs "standing" animation rows in the astronaut sprite sheet requires human visual confirmation.
