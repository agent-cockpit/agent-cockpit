# Phase 16.3 — Collision System: Map Limits, Walls & Object Physics

## Goal

The player cannot walk through walls (terrain ID 3) or outside the walkable
map area (void terrain IDs 2, 7). The player also cannot walk through solid
map objects (desks, couch, piano, etc.) placed via the `objects/manifest.json`.
Movement uses axis-separated AABB slide — the player glides along walls rather
than stopping dead.

---

## Research

### Game collision fundamentals (top-down RPGs)

Top-down games typically use one of three approaches:

| Approach | How it works | Best for |
|---|---|---|
| Tile-grid collision | Blocked tiles = impassable. Convert player pixel pos to tile, check. | Maps with tile-aligned walls. Simple, O(1). |
| AABB (axis-aligned bounding box) | Player has a rectangle; objects have rectangles. Overlap = collision. | Furniture, NPCs, arbitrarily-placed objects. |
| Physics engine | Full rigid-body simulation. | Overkill for a top-down office game. |

**This project needs both:** tile-grid for walls/terrain AND AABB for map objects.

### Axis-separated movement (the key technique)

Naïve collision: try full movement vector → blocked → stop completely.
Result: player freezes at corners, can't slide along diagonal walls. Feels terrible.

**Correct approach — separate axes:**
```
1. Try X movement alone: newX = player.x + dx
   Check if blocked. If not blocked: apply.
   If blocked: don't apply X.

2. Try Y movement alone: newY = player.y + dy
   Check if blocked. If not blocked: apply.
   If blocked: don't apply Y.
```

This gives "slide" behavior: walking diagonally into a north wall, the player
automatically slides east/west. This is how Zelda, Stardew Valley, etc. work.

### Player hitbox design

The player sprite is 64×64px but the collision box should be smaller —
specifically the "feet" area only. This is standard in top-down RPGs because:
1. The sprite is drawn from a 3/4 top-down perspective; the top of the sprite
   appears "further away" on the map, but the feet are the actual world-space position.
2. A full 64×64 hitbox feels sluggish and catches on every wall corner.

**Recommended hitbox:**
```
Offset from player.x, player.y (top-left of sprite):
  hitbox_x = player.x + 12   // centered, with 12px margin each side
  hitbox_y = player.y + 36   // lower 28px of sprite
  hitbox_w = 40
  hitbox_h = 24
```

Export as a constant `PLAYER_HITBOX = { offsetX: 12, offsetY: 36, w: 40, h: 24 }`.

Tunable in a single place — adjust after visual testing.

### Terrain data: what's walkable

From `map.json` terrains and `terrain-map.json`:

| ID | Name | Walkable |
|----|------|---------|
| 0  | (default — outside map area) | NO (void) |
| 1  | dark outer hull floor (interior floor) | YES |
| 2  | deep space void exterior | NO (you'd fall into space) |
| 3  | solid interior wall | NO (it's a wall) |
| 4  | guide line floor | YES |
| 5  | Scuffed Panels (worn floor) | YES |
| 6  | Cracked & Hazardous | YES (dangerous but traversable) |
| 7  | deep space void exterior (variant) | NO |

Blocked terrain IDs: `{0, 2, 3, 7}`.
Walkable terrain IDs: `{1, 4, 5, 6}`.

### Terrain-map coordinate system

`terrain-map.json` uses tile-space coordinates with origin at (-46, -43):

```
tile (tx, ty) → pixel (px, py):
  px = (tx - TILE_ORIGIN_X) * TILE_SIZE = (tx + 46) * 32
  py = (ty - TILE_ORIGIN_Y) * TILE_SIZE = (ty + 43) * 32

pixel (px, py) → tile (tx, ty):
  tx = Math.floor(px / 32) - 46
  ty = Math.floor(py / 32) - 43
```

Tile grid dimensions: 101×101 (x: -46..+54, y: -43..+57).

**Flat array index formula:**
```
index = (ty + 43) * 101 + (tx + 46)
Array size: 101 * 101 = 10201 entries
```

### Building the collision grid

At startup (when terrain-map.json is loaded), build a `Uint8Array(10201)`:
- Default: 0 = WALKABLE (but wait — defaultTerrain is 0 in the JSON, and terrain 0
  is not defined in map.json → treat undefined terrain as BLOCKED)
- Actually: `defaultTerrain: 0` in the JSON + terrain 0 not in the terrains array →
  this means cells not in the sparse list use terrain 0 (undefined/void) → BLOCKED.
- For each cell in `cells[]`: if terrainId in {2, 3, 7} → set bit = BLOCKED (1).
  If terrainId in {1, 4, 5, 6} → set bit = WALKABLE (0).
- Default (terrain 0, not in cells) → BLOCKED (1).

```ts
// CollisionMap.ts
export class CollisionMap {
  private grid: Uint8Array  // 1 = blocked, 0 = walkable
  readonly TILE_W = 101
  readonly TILE_H = 101
  readonly TILE_ORIGIN_X = -46
  readonly TILE_ORIGIN_Y = -43
  readonly TILE_SIZE = 32

  constructor() {
    // Default everything to blocked (void/undefined tiles)
    this.grid = new Uint8Array(101 * 101).fill(1)
  }

  load(terrainMapJson: TerrainMap): void {
    const WALKABLE_IDS = new Set([1, 4, 5, 6])
    for (const cell of terrainMapJson.cells) {
      const idx = this.tileToIndex(cell.x, cell.y)
      if (idx < 0) continue
      this.grid[idx] = WALKABLE_IDS.has(cell.terrainId) ? 0 : 1
    }
  }

  isBlocked(tx: number, ty: number): boolean {
    const idx = this.tileToIndex(tx, ty)
    if (idx < 0) return true  // out of bounds = blocked
    return this.grid[idx] === 1
  }

  pixelToTile(px: number, py: number): { tx: number; ty: number } {
    return {
      tx: Math.floor(px / this.TILE_SIZE) + this.TILE_ORIGIN_X,
      ty: Math.floor(py / this.TILE_SIZE) + this.TILE_ORIGIN_Y,
    }
  }

  private tileToIndex(tx: number, ty: number): number {
    const col = tx - this.TILE_ORIGIN_X  // 0..100
    const row = ty - this.TILE_ORIGIN_Y  // 0..100
    if (col < 0 || col >= 101 || row < 0 || row >= 101) return -1
    return row * 101 + col
  }
}
```

### Object collision: AABB from manifest.json

`objects/manifest.json` has 30 objects. Each has:
```json
{
  "boundingBox": { "x": 242, "y": 666, "width": 92, "height": 92 }
}
```

The coordinates appear to be in **world pixel space** (0–3232 range).
Verification: first object at (242, 666) → tile (242/32 - 46, 666/32 - 43) =
tile (-38.4, -22.2) — that's inside the map bounds. Reasonable for a character
placed near the map's interior. The width/height of 92px ≈ 2.9 tiles, appropriate
for a character or desk sprite.

**IMPORTANT — not all objects should be solid:**
Looking at the manifest, the first object is "Astronaut" (a decorative background
character sprite), not furniture. Only furniture/structural objects should be solid.
Strategy: mark an object as solid if its `name` matches furniture keywords OR
if it's not a character type. Alternatively, load ALL objects as collidable for
now and adjust after visual testing.

Simple approach: treat all objects with `isProgress: false` as potentially solid.
Character objects (name contains "Astronaut", "Robot", "Alien", etc.) should NOT
block movement — they're decorative. Furniture/structural objects (desks, piano,
couch, terminals) SHOULD block.

**Filter heuristic:**
```ts
const CHARACTER_NAMES = /astronaut|robot|alien|hologram|monkey|caveman|ghost|ninja|pirate|medicine/i
const isSolid = !CHARACTER_NAMES.test(obj.name)
```

Or: add a `solid: boolean` field to the manifest (preferred — explicit).

**Reducing AABB to a walkable "collision footprint":**
Object bounding boxes are 64–92px wide. Use a slightly inset hitbox for objects
to allow the player to get close to the edge without bouncing off from far away:
```ts
const OBJECT_INSET = 4  // shrink each side by 4px
```

### Sweep vs discrete collision (for this project's speed)

Player speed is 120px/s. At 60fps, player moves ≤ 2px per frame.
Tile size is 32px. The player CANNOT skip over a tile in one frame.

**Discrete collision is safe here.** No sweep needed.
This simplifies implementation significantly — just check destination tile(s).

### Multi-tile coverage

When the player's hitbox spans multiple tiles (hitbox is 40×24px = ~1.25×0.75 tiles),
check ALL tiles that the hitbox overlaps, not just the center tile.

```ts
function isBBBlocked(
  collisionMap: CollisionMap,
  x: number, y: number,  // top-left of hitbox in world pixels
  w: number, h: number,
): boolean {
  const tileLeft   = Math.floor(x / 32) - 46
  const tileRight  = Math.floor((x + w - 1) / 32) - 46
  const tileTop    = Math.floor(y / 32) - 43
  const tileBottom = Math.floor((y + h - 1) / 32) - 43

  for (let ty = tileTop; ty <= tileBottom; ty++) {
    for (let tx = tileLeft; tx <= tileRight; tx++) {
      if (collisionMap.isBlocked(tx, ty)) return true
    }
  }
  return false
}
```

### Plan 16.3-02: Object collision list

**New file:** `packages/ui/src/game/ObjectCollider.ts`

```ts
export interface CollisionRect { x: number; y: number; w: number; h: number }

export function buildObjectRects(manifest: ObjectManifest): CollisionRect[] {
  const CHARACTER_NAMES = /astronaut|robot|alien|hologram|monkey|caveman|ghost|ninja|pirate|medicine/i
  return manifest.objects
    .filter(obj => obj.visible && !CHARACTER_NAMES.test(obj.name))
    .map(obj => ({
      x: obj.boundingBox.x,
      y: obj.boundingBox.y,
      w: obj.boundingBox.width,
      h: obj.boundingBox.height,
    }))
}

export function isCollidingWithObjects(
  rects: CollisionRect[],
  hitX: number, hitY: number,
  hitW: number, hitH: number,
): boolean {
  for (const r of rects) {
    if (
      hitX < r.x + r.w &&
      hitX + hitW > r.x &&
      hitY < r.y + r.h &&
      hitY + hitH > r.y
    ) return true
  }
  return false
}
```

**Tests:**
1. Character objects are excluded from rect list
2. Furniture objects are included
3. AABB overlap detection: overlapping rects return true
4. Non-overlapping rects return false
5. Edge-touching (not overlapping) returns false

### Plan 16.3-03: Wire collision into movePlayer

**Files to change:**
- `packages/ui/src/game/PlayerInput.ts` — update `movePlayer()` signature
  to accept `collisionMap` and `objectRects`; implement axis-separated collision

**New signature:**
```ts
import type { CollisionMap } from './CollisionMap.js'
import type { CollisionRect } from './ObjectCollider.js'
import { isCollidingWithObjects } from './ObjectCollider.js'

export const PLAYER_HITBOX = { offsetX: 12, offsetY: 36, w: 40, h: 24 }

export function movePlayer(
  player: { x: number; y: number; direction: string; animTime: number },
  keys: ReadonlySet<string>,
  deltaMs: number,
  collisionMap?: CollisionMap,
  objectRects?: CollisionRect[],
): void {
  // ... compute dx, dy as before ...

  // Axis-separated collision
  const hox = PLAYER_HITBOX.offsetX
  const hoy = PLAYER_HITBOX.offsetY
  const hw = PLAYER_HITBOX.w
  const hh = PLAYER_HITBOX.h

  function isSolid(px: number, py: number): boolean {
    const hx = px + hox
    const hy = py + hoy
    if (collisionMap?.isBBBlocked(hx, hy, hw, hh)) return true
    if (objectRects && isCollidingWithObjects(objectRects, hx, hy, hw, hh)) return true
    return false
  }

  // Try X
  const newX = Math.max(0, Math.min(player.x + dx * dist, WORLD_W - 64))
  if (!isSolid(newX, player.y)) {
    player.x = newX
  }

  // Try Y
  const newY = Math.max(0, Math.min(player.y + dy * dist, WORLD_H - 64))
  if (!isSolid(player.x, newY)) {
    player.y = newY
  }

  // ... direction + animTime update ...
}
```

**Backward compatibility:** `collisionMap` and `objectRects` are optional params.
If not provided, movement works as before (no collision). This keeps existing
tests passing without modification.

**New tests:**
1. Player stops when walking into a blocked tile (X axis)
2. Player stops when walking into a blocked tile (Y axis)
3. Player slides along a wall (one axis free, one blocked)
4. Player stops at object bounding box boundary
5. Player is not blocked when collisionMap is not provided (backward compat)

## Coordinate System Clarification (critical)

The terrain-map.json uses tile coordinates. The objects/manifest.json uses pixel
coordinates. Both must be unified into the same world-pixel space before collision:

```
terrain-map cell {x: 0, y: 0}
  → pixel (0 + 46) * 32, (0 + 43) * 32
  → pixel (1472, 1376)  ← player start position ✓

manifest object {boundingBox: {x: 242, y: 666}}
  → already in pixel space (0..3232 range)
```

No transform needed for object rects. Terrain cells need the tile→pixel formula.
The `isBBBlocked()` method works in pixel space, converting internally via
`pixelToTile()` before querying the grid.
