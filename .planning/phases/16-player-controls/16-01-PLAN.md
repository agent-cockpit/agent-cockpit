  <name>Task 1: Create PlayerInput.ts with held-key Set and movePlayer() pure function</name>
  <files>packages/ui/src/game/PlayerInput.ts, packages/ui/src/game/__tests__/PlayerInput.test.ts</files>
  <behavior>
    - movePlayer() with W held: player.x unchanged, player.y decreases by PLAYER_SPEED * (deltaMs/1000)
    - movePlayer() with D held: player.x increases by PLAYER_SPEED * (deltaMs/1000), player.y unchanged
    - movePlayer() with W+D held: both x and y change; total displacement = PLAYER_SPEED * dt (not PLAYER_SPEED * dt * sqrt(2))
    - movePlayer() does not move player below x=0 (left bound)
    - movePlayer() does not move player beyond WORLD_W-64 (right bound)
    - movePlayer() does not move player below y=0 (top bound)
    - movePlayer() does not move player beyond WORLD_H-64 (bottom bound)
    - movePlayer() with D held sets player.direction = 'east'
    - movePlayer() with W+D held sets player.direction = 'north-east'
    - movePlayer() with no keys held does NOT change player.direction
    - onKeyDown is a no-op (does not add to keysDown) when document.activeElement is an HTMLInputElement
    - onKeyDown is a no-op when document.activeElement is an HTMLTextAreaElement
    - attachInput adds keydown and keyup listeners; detachInput removes them
    - detachInput clears the keysDown Set (player stops moving after detach)
  </behavior>
  <action>
    Create packages/ui/src/game/PlayerInput.ts with:

    1. `export const PLAYER_SPEED = 120` (pixels per second)

    2. Module-level `const keysDown = new Set<string>()` and `let _attached = false`

    3. `onKeyDown(e: KeyboardEvent): void` — checks document.activeElement:
       if active is HTMLInputElement, HTMLTextAreaElement, HTMLSelectElement, or HTMLElement with isContentEditable, return early.
       Otherwise: keysDown.add(e.code).
       Call e.preventDefault() only for ArrowUp, ArrowDown, ArrowLeft, ArrowRight.

    4. `onKeyUp(e: KeyboardEvent): void` — keysDown.delete(e.code)

    5. `export function attachInput(): void` — guard with _attached flag, add listeners to window, set _attached = true

    6. `export function detachInput(): void` — remove listeners, keysDown.clear(), _attached = false

    7. `export function getKeysDown(): ReadonlySet<string>` — return keysDown

    8. `export function movePlayer(player: { x: number; y: number; direction: string }, keys: ReadonlySet<string>, deltaMs: number): void`:
       - dt = deltaMs / 1000
       - dist = PLAYER_SPEED * dt
       - up = keys.has('KeyW') || keys.has('ArrowUp')
       - down = keys.has('KeyS') || keys.has('ArrowDown')
       - left = keys.has('KeyA') || keys.has('ArrowLeft')
       - right = keys.has('KeyD') || keys.has('ArrowRight')
       - dx/dy from up/down/left/right booleans
       - if dx !== 0 && dy !== 0: multiply both by 0.7071 (INV_SQRT2)
       - clamp player.x to [0, WORLD_W - 64], player.y to [0, WORLD_H - 64]
       - call internal `deriveDirection(dx, dy)` — if non-null, set player.direction

    9. Internal `deriveDirection(dx, dy)` — maps all 8 combinations of dx/dy signs to Direction strings.
       Returns null when dx === 0 && dy === 0.

    Import WORLD_W, WORLD_H from './GameState.js'
    Import Direction from '../components/office/spriteStates.js'

    Write tests FIRST (RED), then implement (GREEN). Test file at packages/ui/src/game/__tests__/PlayerInput.test.ts.
    Tests call movePlayer() directly with a plain object player and a Set of key codes.
    For event listener tests, use vi.spyOn(window, 'addEventListener') and check document.activeElement via vi.spyOn(document, 'activeElement', 'get').
  </action>
