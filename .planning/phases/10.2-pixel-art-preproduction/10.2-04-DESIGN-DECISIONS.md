## Q1 — Space Sub-Theme: Space Station Ops

**Choice: A — Space Station Ops**

Visual identity: ISS/mission control aesthetic. Functional, operational, slightly military.
- Backgrounds: dark gray/charcoal base with warm grid lines
- Accent colors: NASA orange (`#FF6B2B`), instrument blue (`#58A6FF`), console gray (`#8B9DB8`)
- Lighting: cool fluorescent overhead, warm amber alert states
- References: FTL crew sprites, real NASA mission control rooms, Dead Cells functional UI

The space station grid floor (Q5) anchors this — agents are literally crew members working at their stations.

Assign character by hashing sessionId modulo 10. Same session always gets same character — stable across page reloads.

```typescript
const CHARACTER_TYPES = ['astronaut','robot','alien','hologram','monkey','caveman','ghost','ninja','pirate','medicine-woman']
const characterIndex = parseInt(sessionId.slice(-4), 16) % CHARACTER_TYPES.length
const characterType = CHARACTER_TYPES[characterIndex]
```

Sprite sheet path pattern: `/sprites/{character-type}-sheet.png`

## Q3 — Size and Directions: 64px, 8 Directions

**Choice: 64px sprites, 8 directions**

8 directions: N, NE, E, SE, S, SW, W, NW

Rationale: The Office Mode canvas allows dragging — 8 directions lets the character face the direction they're being dragged toward. At 64px, facial expressions and costume details read clearly.

**Impact on current code:**
- `AgentSprite.tsx` currently renders `width: 32, height: 32` → update to 64×64
- CSS `background-position` offsets will use 64px grid
- Sprite sheet layout: rows = states, columns = directions (8 cols × 64px = 512px wide per state row)

## State Animation Specification (per character, per state)

### Unique animation states (3 states × 8 directions × 10 characters = 240 animation gens)

```
blocked:
  tool: animate_character
  template: "alert" (fallback: "hurt", then custom "arms-up alarmed")
  action_description: "arms raised in alarm, urgent shaking motion, distressed expression"
  directions: [N, NE, E, SE, S, SW, W, NW]

completed:
  tool: animate_character
  template: "victory" (fallback: "cheer", then "jump")
  action_description: "victory celebration, arms raised in triumph, happy expression"
  directions: [N, NE, E, SE, S, SW, W, NW]

failed:
  tool: animate_character
  template: "collapse" (fallback: "hurt", then "idle-sad")
  action_description: "slumped defeated posture, shoulders drooped, exhausted"
  directions: [N, NE, E, SE, S, SW, W, NW]
```

### Color states (5 states) — use idle animation + CSS glow overlay
No additional animation generation needed for: planning, coding, reading, testing, waiting.
CSS `box-shadow` or `filter: drop-shadow()` applies the state color glow.
