Execution steps (identical to 28-01):

1. **Load auth token** from `~/.claude.json` → `mcpServers.pixellab.env.PIXELLAB_SECRET_KEY` (in-process; never log).
2. **For each character sequentially:**
   a. POST `https://api.pixellab.ai/v2/animate-character` with body `{ character_id, template_animation_id: "walking-8-frames", directions: DIRECTIONS }`.
   b. Template fallback chain on `422`: `walking-8-frames` → `walk` → `walking`. On `423`, sleep 60s and retry up to 3 times.
   c. Poll all 8 `background_job_ids` at 45s intervals via `GET /v2/background-jobs/{id}` until all `status === 'completed'`. Abort on `status === 'failed'`.
   d. For each completed job's `last_response.images[]` (expect 8), decode base64 → `sharp.raw({width, height, channels:4}).png().toFile(...)`:
      ```js
      const dir = path.join(ROOT, 'assets', 'raw', character.name, 'walk', DIRECTIONS[d])
      await fs.promises.mkdir(dir, { recursive: true })
      const buffer = Buffer.from(image.base64, 'base64')
      await sharp(buffer, { raw: { width: image.width, height: image.height, channels: 4 } })
        .png()
        .toFile(path.join(dir, `frame-${i}.png`))
      ```
      DO NOT resize here — keep native 92×92. `build-spritesheet.ts` handles the 64×64 resize in Plan 28-03.
   e. Log `[{character}] {direction}: saved 8 frames` (no secrets, no base64).
   f. Move to next character. Sequential only.

3. **Verify on disk:** count PNG files per direction per character — must be 8 each. Total = 5 × 8 × 8 = 320.
