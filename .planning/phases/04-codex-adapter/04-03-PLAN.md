CodexAdapter class contract:
```typescript
export class CodexAdapter {
  constructor(
    sessionId: string,
    workspacePath: string,
    db: Database.Database,
    onEvent: (event: NormalizedEvent) => void,
    threadId?: string  // if provided, call thread/resume instead of thread/start
  )
  start(): Promise<void>  // spawn + initialize handshake + thread/start or resume
  resolveApproval(approvalId: string, decision: 'approve' | 'deny' | 'always_allow'): void
  stop(): void            // kill child process, clean up
}
```

approvalQueue.decide() currently calls `resolveApproval` from hookServer (Claude-specific). The Codex adapter needs to register its own resolver. Pattern: pass a `resolverFn: (approvalId, decision) => void` to a registry. The cleanest approach for Phase 4: add a `resolveCodexApproval` module-level function to codexAdapter.ts and call it from `approvalQueue.decide()` after checking provider — OR register the resolver in a shared Map keyed by approvalId at approval registration time. Use the Map-per-approvalId approach: store `{ resolve: fn }` alongside the pending event. This avoids modifying approvalQueue.ts.

The ApprovalQueue.decide() currently calls `resolveApproval` (Claude hook server) always. To support Codex without modifying ApprovalQueue, register a per-approvalId resolver in a module-level Map in codexAdapter.ts and export a `resolveCodexApproval(approvalId, decision)` function. Then update `approvalQueue.decide()` to check both resolvers (call resolveApproval which no-ops if not found, then call resolveCodexApproval which no-ops if not found).

    ```typescript
    import { spawn, type ChildProcess } from 'node:child_process';
    import readline from 'node:readline';
    import { randomUUID } from 'node:crypto';
    import type Database from 'better-sqlite3';
    import type { NormalizedEvent } from '@cockpit/shared';
    import { parseCodexLine, type CodexParserContext } from './codexParser.js';
    ```

    Key implementation points (from RESEARCH.md — anti-patterns to avoid):

    1. **Request ID counter:** Module-level `let requestIdCounter = 1`. Use `nextId()` → `requestIdCounter++`. Never reuse IDs.

    2. **pendingRequests Map:** `Map<number, (result: unknown) => void>` for correlating our outgoing requests (initialize, thread/start, thread/resume).

    3. **pendingCodexApprovals Map:** `Map<number, { approvalId: string; timer: NodeJS.Timeout }>` keyed by Codex server integer id. Reverse lookup in `resolveApproval`.

    4. **Stdin guard:** All writes: `if (!this.proc.killed && this.proc.stdin?.writable) { this.proc.stdin.write(JSON.stringify(msg) + '\n'); }`.

    5. **Readline loop:** Three branches per line (from RESEARCH.md Pattern 1):
       - Has `id` + `result` → response to our request → call pendingRequests callback
       - Has `method` + no `id` → server notification → call `parseCodexLine`, emit result if non-null
       - Has `method` + has `id` → server-initiated approval request → call `handleServerRequest`

    6. **handleServerRequest:** Call `parseCodexLine` which returns `approval_request` event with `_codexServerId`. Store in `pendingCodexApprovals`. Call `onEvent(event)` + `approvalQueue.register(event.approvalId, event, db)`. Set timeout for auto-deny.

    7. **resolveApproval:** Reverse-lookup `pendingCodexApprovals` by approvalId. Map decision: `approve/always_allow` → `'accept'`/`'acceptForSession'`, `deny` → `'decline'`. Write reply to stdin.

    8. **Export resolveCodexApproval:** Module-level `Map<string, (decision) => void>` called `codexApprovalResolvers`. `resolveCodexApproval(approvalId, decision)` calls and deletes from the Map. Register in this Map when adapter handles a server approval request.

    9. **Process exit cleanup:** `proc.on('exit', () => { this.proc = null; clean up pending approvals })`.

    10. **ENOENT handling:** Wrap `spawn` in try/catch or listen to `proc.on('error')`. If `err.code === 'ENOENT'`, emit a `provider_parse_error` with errorMessage "codex binary not found on PATH" and do NOT crash.

    11. **Session resume:** In `start()`, query DB for existing `thread_id`. If found, call `thread/resume`; if `thread/resume` returns error, log warning and fall back to `thread/start`.
