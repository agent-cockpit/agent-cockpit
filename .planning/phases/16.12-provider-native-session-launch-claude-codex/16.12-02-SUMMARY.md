## Decisions Made

- Settings file written to `os.tmpdir()/cockpit-claude-{uuid}.json` — one file per session, no collisions between concurrent sessions
- `hookParser.ts` required no code changes: pre-registering via `setClaudeSessionId(db, sessionId, sessionId)` before spawn means Tier 2 lookup always finds it when hooks arrive
- `proc.once('spawn')` resolves launch() immediately when OS starts the process — does not wait for process exit
- ENOENT on spawn error → `MISSING_BINARY` (consistent with preflight's MISSING_BINARY); other errors → `SPAWN_FAILED`
- EventEmitter mock needs `proc.on('error', noop)` guard to prevent unhandled Node rejection when tests end before event fires
