**OfficePage test regression detail:** The test "canvas click on NPC teleports camera to centre on that NPC" sets `viewportW: 800` and fires a click at screen `(420, 320)` expecting world coords `(420, 320)`. After the zoom correction in Plan 03, `handleClick` computes `worldX = (420/2) + 0 = 210`, which misses the NPC at `(400, 300)`. The test was only partially updated in commit `e2fafaa` (added `zoom: 2` to camera literal) but was not updated to account for the zoom division in the click coordinate calculation. This is a test suite regression introduced by Phase 16.1 Plan 03 — the production code is correct (the zoom correction is the intended behavior), but the test's click coordinates and expected values need updating to match the new coordinate math.
