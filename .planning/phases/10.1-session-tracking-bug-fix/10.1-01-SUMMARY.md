## Decisions Made

- **claude_id must be UNIQUE NOT NULL:** The plan schema showed `claude_id TEXT NOT NULL` but the test expected `INSERT OR IGNORE` to prevent a second row for the same `claude_id`. Since `session_id` is the PK, two rows with different `session_id` but same `claude_id` would both insert without UNIQUE. Added `UNIQUE` constraint so the first write wins.
- **Replaced require() stubs:** Wave 0 test stubs used `require('../db/queries.js')` inside test bodies as a placeholder mechanism that was meant to fail. With the functions now exported, replaced with the already-present top-level ES module import (line 3 of database.test.ts).
