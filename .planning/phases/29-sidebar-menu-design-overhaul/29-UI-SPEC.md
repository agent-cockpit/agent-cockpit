## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual Tailwind v4 tokens) |
| Preset | not applicable |
| Component library | none (custom) |
| Icon library | none detected — use text glyphs only (`[<]`, `[>]`, `[X]`, `>>`) |
| Font — display | `"Press Start 2P"` → `"VT323"` → `"Courier New"` → monospace (declared in `--font-sidebar-display`) |
| Font — body | `"IBM Plex Sans"` → `"Inter"` → `"Segoe UI"` → sans-serif (declared in `--font-sidebar-body`) |
| Font — data/mono | `"IBM Plex Mono"` → `"JetBrains Mono"` → `"Courier New"` → monospace (declared in `--font-mono-data`) |

> Note: fonts are declared as CSS custom property stacks in `packages/ui/src/index.css` — they are NOT loaded from Google Fonts or via `@font-face`. If Press Start 2P or IBM Plex are not installed locally they will fall back to the next stack item. Phase 29 MUST add a `<link rel="preconnect">` + Google Fonts import for `Press Start 2P` and `IBM Plex Mono` in `packages/ui/index.html` as part of implementation so the game-UI aesthetic is actually visible.

## Spacing Scale

Tailwind v4 uses the default spacing scale (multiples of 0.25rem / 4px at 16px root). All component spacing below uses this scale. Game UI conventions favour compact density — no generous padding.

| Token | px value | Use |
|-------|----------|-----|
| `p-1` / `gap-1` | 4px | Internal micro-gaps (dot + label) |
| `p-2` / `gap-2` | 8px | Row internal padding (face card to text) |
| `p-3` / `gap-3` | 12px | Section padding (sidebar header, menu sections) |
| `p-4` | 16px | Modal/menu content outer padding |
| `py-2.5` | 10px | Session row vertical padding (preserve existing) |
| `px-3` | 12px | Session row horizontal padding (preserve existing) |
| `gap-1.5` | 6px | Status dot + label gap |
| `gap-3` | 12px | Face avatar + text column gap |

Sidebar header bracket frame: `px-3 py-2.5` (preserve from `OpsLayout.tsx`).
Session row: `px-3 py-2.5` (preserve from `MapSidebar.tsx`).
Menu modal content: `p-4` with `space-y-4` between sections (preserve from `MenuPopup.tsx`).

## Implementation Notes

### Phase 16.8 Constraints — MUST BE PRESERVED

These decisions are locked and must not be changed in Phase 29:

1. **Row click contract**: `selectSession(sessionId)` MUST execute BEFORE `onFocusSession(sessionId)` in every session row click handler. Order is required for correct state sequencing.
2. **Status mapping**: `SessionStatus` has exactly three values: `active`, `ended`, `error`. No additional statuses may be introduced.
3. **Secondary metadata line**: conditional — only rendered when `pendingApprovals > 0 || status !== 'active'`. Never render when session is active with no approvals.
4. **Sort order**: `[...sessions].sort((a, b) => b.lastEventAt.localeCompare(a.lastEventAt))` — local sort, most-recent first, must be preserved.
5. **Resize handle**: desktop-only (`matchMedia('(min-width: 1024px)')`), width clamp `[260, 460]px`, stored in `localStorage` via `'cockpit.sidebar.width'` key.
6. **Camera snap on sidebar focus**: both `camera.targetX/Y` AND `camera.x/Y` (current) must be set together during `onFocusSession` to preserve immediate snap without lerp delay.

### Token Naming Conventions

- All new tokens go into the `@theme` block in `packages/ui/src/index.css` (Tailwind v4 convention)
- Token names follow the `--color-cockpit-*` namespace for game-palette colors
- Do not create tokens for values that appear only once — inline those as OKLCH literals
- Do not rename existing tokens — backward compat with all existing components

### Font Loading (Required Pre-condition for Phase 29)

Add to `packages/ui/index.html` `<head>` before any content:
```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
```
