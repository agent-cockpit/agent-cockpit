# Phase 16.7 Plan 01: CollisionMap TDD — Tile AABB + Object Registry Summary

**One-liner:** CollisionMap built via TDD — sparse tile Set + pixel-space object AABB registry for solid-wall and furniture collision queries.

## What Was Built

`CollisionMap.ts` provides the core data structure for phase 16.7 collision physics:

- `loadTerrain(data)` — iterates sparse terrain-map.json cells, adds tile keys `"${tileX},${tileY}"` to a `Set<string>` for terrainId 2 (void exterior) and 3 (interior wall) only
- `loadObjects(objects)` — filters manifest.json objects, skips `Character:` descriptions and `visible: false`, registers remaining bounding boxes as `Rect[]`
- `overlaps(x, y, w, h)` — tests 4 AABB corners against solidTiles Set (O(1) each), then AABB-vs-AABB against solidObjects array
- `PLAYER_HITBOX` exported: `{ offsetX: 16, offsetY: 32, w: 32, h: 32 }` — foot-area hitbox for plan 02

## Key Decisions

1. **Sparse Set for solid tiles** — `Set<string>` keyed by `"tileX,tileY"` in tile-space. Map has ~4428 solid cells out of 10201+ possible tiles. Sparse set avoids allocating a full 2D array and gives O(1) lookup.

2. **Character objects are non-solid** — Objects whose `description` starts with `"Character:"` (astronauts, robots, aliens from map editor layers 2–11) are ambient art, not collidable obstacles. Interactive NPCs are tracked in `gameState.npcs`.

3. **4-corner AABB test for tiles** — Checking all 4 corners of the player hitbox covers all standard collision scenarios at tile sizes (32px) that are larger than per-frame movement (~2px at 60fps).

4. **PLAYER_HITBOX in CollisionMap.ts** — Co-located with the collision module so plan 02 imports one module for both the CollisionMap class and hitbox dimensions.
